# 📖 오늘 배운 내용

---

> **서버가 느려지거나 해킹이 의심되는 상황에서 `시스템 자원`과 `네트워크 상태`를 진단하여 즉각 대처하는 법을 익혔으며, `패키지 관리`부터 `서비스 제어`, `디스크 용량 확인` 및 `파일 압축`까지 리눅스 서버 운영 전반에 걸친 필수 관리 명령어들을 종합적으로 실습했다.**

## 💡 Why & How | 핵심 개념
* **시스템 소프트웨어 관리:** 필요한 도구를 설치(`apt`)하고, 백그라운드에서 돌아가는 서비스들이 죽지 않게 관리(`systemctl`)하는 법을 익혔다. 서버 운영의 가장 기본이 되는 '심장' 관리법이다.
* **파일 및 디스크 정밀 진단:** 단순히 파일을 찾는 것(`find`)을 넘어, 파일의 상세 상태(`stat`)와 서버 전체의 남은 용량(`df`, `du`)을 체크하여 시스템 마비를 사전에 방지하는 법을 배웠다.
* **리소스 및 네트워크 모니터링:** CPU/메모리 자원을 누가 쓰는지(`top`, `ps`) 감시하고, 열려 있는 통로(`netstat`)를 확인하여 보안 위협에 대처했다.
* **효율적인 작업 환경:** 중요 데이터를 백업하는 압축(`tar`) 기술과, 수정한 설정을 재부팅 없이 즉시 적용하는 `source` 명령어의 편리함을 확인했다.

## 🛠️ 오늘 사용한 핵심 명령어

```bash
# 1. 패키지 및 서비스 관리 (Package & Service)
sudo apt update && sudo apt upgrade  # 패키지 목록 업데이트 및 업그레이드
sudo systemctl start [서비스명]        # 특정 서비스 실행 (stop/restart/status 가능)
sudo systemctl enable [서비스명]       # 부팅 시 자동 실행 설정

# 2. 파일 검색 및 상세 정보 (Search & Stat)
find /etc -name "*.conf"             # /etc 폴더 내에서 .conf 파일 찾기
stat [파일명]                         # 파일의 생성/수정/변경 시간 및 상세 정보 확인

# 3. 디스크 및 리소스 관리 (Disk & Resource)
df -h                                # 전체 디스크 남은 용량 확인 (Human-readable)
du -sh [경로]                         # 특정 폴더/파일이 차지하는 용량 합계
top                                  # 실시간 CPU/메모리 점유율 모니터링
ps -ef | grep [프로세스명]             # 실행 중인 특정 프로세스 상세 검색
sudo kill -9 [PID]                   # 악성 프로세스 즉시 강제 종료

# 4. 압축 및 환경 설정 (Archive & Shell)
tar -cvf backup.tar [폴더]            # 파일/폴더를 하나로 묶기
tar -xvf backup.tar                  # 묶인 파일 풀기
netstat -tuln                        # 열려 있는 네트워크 포트 목록 확인
source ~/.bashrc                     # 변경된 쉘 설정 파일 즉시 적용
```
