# 📝 기본 개발환경 구축 

>**과목:** 1과목. 사무자동화시스템  
**단원:** 1. 개발자 환경 구축 > 2. 기본 개발환경 구축  
**학습 목표:** 소프트웨어 개발에 필요한 도구들의 종류와 역할을 이해하고, 실제 환경 구축을 위한 패키지 및 라이브러리 관리 방법을 숙지한다.

---

## 📍 1. 개발도구 종류

개발자가 작성한 소스 코드(인간의 언어)가 컴퓨터가 이해할 수 있는 기계어(0과 1)로 변환되어 실행되기까지는 여러 가지 도구의 협업이 필요합니다. 각 도구의 역할을 명확히 구분하는 문제가 자주 출제됩니다.

### 1) 핵심 개발도구의 역할 (코드의 실행 흐름)
1. **에디터 (Editor):**
   * 프로그래머가 소스 코드를 작성하고 편집하는 텍스트 편집기입니다. (예: 메모장, VS Code, Sublime Text)
2. **컴파일러 (Compiler) / 인터프리터 (Interpreter):**
   * **컴파일러:** 작성된 소스 코드 전체를 한 번에 기계어(목적 프로그램)로 번역합니다. (C, Java)
   * **인터프리터:** 소스 코드를 한 줄씩 읽어서 바로 실행합니다. 번역 속도는 빠르나 실행 속도는 컴파일러보다 느립니다. (Python, JavaScript)
3. **링커 (Linker):**
   * 컴파일된 여러 개의 목적 파일(Object File)과 라이브러리를 연결하여 하나의 **실행 가능한 파일(.exe 등)**로 만들어 주는 도구입니다.
4. **로더 (Loader):**
   * 실행 가능한 파일을 주기억장치(RAM)에 적재(Load)하여 CPU가 실행할 수 있도록 해주는 프로그램입니다.
   * **로더의 4대 기능 (빈출 ⭐):** 할당(Allocation) ➔ 연결(Linking) ➔ 재배치(Relocation) ➔ 적재(Loading)
5. **디버거 (Debugger):**
   * 프로그램 내에 존재하는 논리적 오류(Bug)를 찾아내고 수정(Debug)할 수 있도록 돕는 도구입니다.

### 2) 통합 개발 환경 (IDE - Integrated Development Environment)
* 위에서 언급한 에디터, 컴파일러, 디버거 등의 도구들을 하나로 묶어, 하나의 프로그램 안에서 모든 개발 작업을 처리할 수 있도록 만든 소프트웨어입니다.
* **대표적인 IDE:** Visual Studio, Eclipse, IntelliJ IDEA, PyCharm.

## 📍 2. 프로그래밍 개발도구 설치

실제 개발 환경을 구축할 때는 사용하는 언어의 특성에 맞게 필수 도구들을 설치하고 환경 변수를 설정해야 합니다.

### 1) 주요 언어별 필수 설치 도구
* **Java (자바):**
  * **JDK (Java Development Kit):** 자바 개발을 위한 필수 도구 모음 (컴파일러 `javac` 포함).
  * **JRE (Java Runtime Environment):** 자바 프로그램을 실행하기 위한 환경. (JDK를 설치하면 JRE도 함께 설치됨)
  * **JVM (Java Virtual Machine):** 운영체제에 상관없이 자바 프로그램이 실행될 수 있도록 해주는 가상 머신 (플랫폼 독립성 제공).
* **C / C++:**
  * **GCC (GNU Compiler Collection):** 리눅스 및 오픈소스 진영에서 가장 널리 쓰이는 표준 C/C++ 컴파일러.
* **Python (파이썬):**
  * 파이썬 공식 인터프리터 설치 필요.

### 2) 환경 변수 (PATH) 설정
* 개발도구를 설치한 후, 운영체제의 `PATH` 환경 변수에 해당 도구의 실행 파일 경로(`\bin` 디렉토리 등)를 추가해야 합니다.
* **설정 이유:** 이 작업을 거쳐야만 명령 프롬프트(CMD)나 터미널의 **어느 디렉토리(위치)에서든** 컴파일러(`javac`, `python` 등) 명령어를 즉시 실행할 수 있습니다.

## 📍 3. 라이브러리, 패키지 설치

현대의 소프트웨어 개발은 모든 것을 처음부터 짜지 않습니다. 남들이 만들어 둔 유용한 코드 뭉치를 가져와 조립하는 능력이 매우 중요합니다.

### 1) 라이브러리와 패키지의 개념
* **라이브러리 (Library):** 특정 기능들을 수행하는 함수나 클래스들을 모아둔 코드의 집합소입니다. (예: 수학 계산 라이브러리, 그래픽 라이브러리)
* **패키지 (Package):** 서로 관련된 라이브러리와 모듈들을 구조적으로 잘 정리하여 배포하기 좋게 묶어놓은 단위입니다.

### 2) 의존성(Dependency) 문제
* 패키지 A를 설치할 때, 이 패키지가 제대로 동작하기 위해 반드시 패키지 B가 필요한 상황을 '의존성'이라고 합니다.
* 과거에는 이 의존성을 개발자가 일일이 찾아 설치해야 했으나, 현대에는 **패키지 관리자(Package Manager)**가 이를 자동으로 해결해 줍니다.

### 3) 운영체제 및 언어별 주요 패키지 관리자 
각 환경에 맞는 패키지 관리자 명령어를 알아두어야 합니다.

| 환경 / 언어                    | 패키지 관리자 명칭              | 설치 명령어 예시                                                                      |
| :----------------------------- | :------------------------------ | :------------------------------------------------------------------------------------ |
| **Linux (Debian/Ubuntu 계열)** | `APT` (Advanced Packaging Tool) | `sudo apt install [패키지명]`                                                         |
| **Linux (RedHat/CentOS 계열)** | `YUM` 또는 `DNF`                | `sudo yum install [패키지명]`                                                         |
| **Python (파이썬)**            | `pip` (Python Package Index)    | `pip install [패키지명]`                                                              |
| **Node.js (자바스크립트)**     | `npm` (Node Package Manager)    | `npm install [패키지명]`                                                              |
| **Java (자바 - 빌드 도구)**    | `Maven`, `Gradle`               | (프로젝트 설정 파일인 `pom.xml`이나 `build.gradle`에 의존성을 명시하면 자동 다운로드) |
