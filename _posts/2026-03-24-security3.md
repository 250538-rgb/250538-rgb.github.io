# 📖 오늘 배운 내용

---

> **리눅스 시스템의 사용자 계정과 그룹을 생성·수정하고, 더 이상 필요 없는 계정을 안전하게 삭제하는 '계정 생명주기' 전체를 마스터했다. 또한 파일 소유권 변경(chown)과 특수 권한(Sticky Bit)을 통한 공용 폴더 관리 기법을 실습했다.**

## 💡 Why & How | 핵심 개념
* **useradd vs adduser:** `useradd`는 기본 설정만 해주는 불친절한(저수준) 명령어이고, `adduser`는 비번과 홈 디렉토리까지 자동으로 챙겨주는 친절한(고수준) 명령어다. 실수를 줄이기 위해 `adduser`를 권장한다.
**누구의 파일인가? (chown):** 모든 파일은 주인(Owner)이 있다. `chown`으로 소유자와 그룹을 바꿀 수 있으며, 파일 권한 관리의 핵심 도구다.
* **권한 계산법 (chmod):** 읽기(4), 쓰기(2), 실행(1)의 합으로 권한을 준다. `755`면 주인은 다 하고 나머지는 읽고 실행만 가능하다는 뜻이다.
* **사용자 삭제와 흔적 지우기:** 사용자를 삭제할 때 단순히 이름만 지우면 그 사용자가 썼던 파일들이 '주인 없는 파일'로 남아 보안 취약점이 될 수 있다. **`userdel -r`**이나 **`deluser --remove-home`**을 써서 홈 디렉토리까지 깔끔하게 밀어버리는 습관이 중요하다.
* **그룹 관리의 정석 (-aG):** 사용자를 새 그룹에 넣을 때 `-a`를 빼먹으면 기존 그룹에서 쫓겨난다. "Append(추가)"를 뜻하는 `-a`를 반드시 기억해야 한다.
* **스티키 비트(Sticky Bit):** "생성은 자유롭지만, 삭제는 주인만!" 공용 폴더(`/tmp`)에서 남의 파일을 함부로 지우지 못하게 막는 최후의 보루다.
* 

## 🛠️ 오늘 사용한 핵심 명령어

```bash
# 1. 사용자 계정 및 그룹 생성 (Creation)
sudo adduser user1         # [권장] 대화형으로 비번/홈디렉토리 자동 생성
sudo useradd -m user2      # 수동 생성 (-m 옵션이 있어야 홈디렉토리 생성)
sudo groupadd sec_group    # 새로운 그룹(sec_group) 생성

# 2. 계정 정보 수정 및 그룹 관리 (Management)
sudo passwd user1          # 비밀번호 설정 및 변경
sudo usermod -aG sec_group user1 # user1을 위에서 만든 sec_group에 안전하게 추가
sudo chown user1:sec_group file.txt # 파일의 소유자를 user1, 그룹을 sec_group으로 변경

# 3. 사용자 및 그룹 삭제 (Deletion)
sudo userdel -r user1      # 계정과 홈 디렉토리를 한 번에 삭제 (저수준)
sudo deluser --remove-home user1 # [권장] 계정과 홈 디렉토리를 안전하게 삭제 (고수준)
sudo groupdel sec_group    # 그룹 삭제

# 4. 특수 권한 및 보안 점검 (Security)
chmod 1777 /share_dir      # 스티키 비트 설정 (맨 앞의 1)
find / -perm -4000 -type f 2>/dev/null # SetUID가 걸린 위험한 파일 탐색
```
