# 📖 오늘 배운 내용

---

> **가상 머신(VM)을 이용해 Ubuntu(방어자)와 Kali Linux(공격자) 가상 머신을 설치하고, 패키지 최신화 및 고정 IP 설정을 통해 표준 실습 환경을 구축했다.**

## 💡 Why & How | 핵심 개념
* **우리끼리만 통신하는 네트워크:** 외부 인터넷이랑은 연결되면서 가상 머신들끼리만 서로 보일 수 있게 'NAT 네트워크'라는 가상의 선을 연결했다. 이렇게 해야 공격자 PC에서 방어자 서버로 접속할 수 있다.
* **서버 주소 고정하기(IP):** 서버 주소가 매번 바뀌면 나중에 방화벽을 세우거나 공격 타겟을 찾기 힘들다. 그래서 방어용 서버 주소를 `192.168.0.30`으로 고정해두었다.
* **기록의 기준, 시간 맞추기:** 해킹 사고가 터졌을 때 '몇 시에' 들어왔는지 아는 게 제일 중요하다. 그래서 서버 시간을 서울 시간으로 정확하게 맞추는 설정부터 꼼꼼히 챙겼다.

## 🛠️ 오늘 사용한 핵심 명령어
```bash
# 시스템 패키지 리스트 업데이트 및 최신버전 업그레이드
sudo apt update && sudo apt upgrade -y

# 시스템 시간대 설정 및 동기화 상태 확인
sudo timedatectl set-timezone Asia/Seoul
sudo systemctl restart systemd-timesyncd

# 네트워크 설정 적용 및 IP 주소 확인
sudo systemctl restart NetworkManager
ip addr
```
