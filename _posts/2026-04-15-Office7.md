# 📝 데이터베이스 프로그램 활용

>**과목:** 1과목. 사무자동화시스템  
**단원:** 3. 사무자동화프로그램 활용 > 2. 데이터베이스 프로그램 활용  
**학습 목표:** 관계형 데이터베이스(RDBMS)의 구조를 이해하고, 테이블 설계(스키마)부터 데이터 조회(쿼리), 보고서 출력까지의 전체 프로세스를 파악한다.

---

## 📍 1. 스키마 작성 및 변경

데이터베이스의 뼈대인 '스키마'를 설계하고 구조를 정의하는 단계입니다.

### 1.1) 스키마(Schema)의 개념
* 데이터베이스의 구조와 제약 조건에 관한 전반적인 명세를 의미합니다.
* **필드(Field):** 열(Column)에 해당하며, 데이터의 최소 단위입니다.
* **레코드(Record):** 행(Row)에 해당하며, 관련된 필드들의 모임입니다.

### 1.2) 데이터 형식 및 제약 조건
* **기본키(Primary Key):** 레코드를 유일하게 식별할 수 있는 필드입니다. (중복 불가, NULL 값 불가)
* **데이터 형식:** 텍스트, 숫자, 날짜/시간, 일련번호(자동 증가), 예/아니오 등 데이터 성격에 맞게 지정합니다.

---

## 📍 2. 데이터 연결 및 가져오기

외부 파일의 데이터를 액세스 내부로 가져오거나 연결하여 사용하는 방법입니다.

### 2.1) 데이터 가져오기 (Import)
* 외부 파일(엑셀, 텍스트 등)의 데이터를 복사하여 액세스 테이블로 만듭니다. 원본 파일이 수정되어도 액세스 데이터는 변하지 않습니다.

### 2.2) 데이터 연결 (Link)
* 원본 파일과 액세스 테이블을 연결합니다. 원본 파일의 내용이 바뀌면 액세스에서도 실시간으로 반영됩니다.

---

## 📍 3. 테이블 활용

테이블 간의 관계를 설정하여 데이터의 무결성을 유지하는 단계입니다.

### 3.1) 관계 설정 (Relationship)
* 공통된 필드를 기준으로 두 테이블을 연결합니다. (주로 1:N 관계가 가장 많음)
* **참조 무결성 유지:** 기본 테이블에 없는 데이터를 관련 테이블에 입력할 수 없도록 강제하여 데이터의 일관성을 보장합니다.

---

## 📍 4. 데이터 조회 및 집계 (쿼리)

방대한 데이터 속에서 원하는 조건에 맞는 정보만 추출하거나 계산하는 과정입니다.

### 4.1) 선택 쿼리
* 특정 조건(`WHERE` 절)을 만족하는 레코드를 찾아 화면에 표시합니다.
* **와일드카드 활용:** `*` (모든 글자), `?` (한 글자)를 사용하여 패턴 검색이 가능합니다.

### 4.2) 집계 함수 및 요약
* `SUM`(합계), `AVG`(평균), `COUNT`(개수), `MAX/MIN` 등을 활용하여 데이터를 그룹화하고 요약합니다.

---

## 📍 5. 매크로 및 모듈

작업을 자동화하거나 사용자 정의 기능을 추가하는 도구입니다.

### 5.1) 매크로 (Macro)
* 자주 사용하는 동작(폼 열기, 메시지 박스 띄우기 등)을 미리 정의된 액션 목록에서 선택하여 자동화합니다.

### 1.2) 모듈 (Module)
* 매크로보다 복잡한 기능을 구현하기 위해 **VBA(Visual Basic for Applications)** 언어로 직접 코드를 작성하는 단위입니다.

---

## 📍 6. 보고서 작성 및 활용

분석된 데이터를 종이에 출력하거나 화면에 보여주기 위해 가독성 있게 꾸미는 단계입니다.

### 6.1) 보고서 구역 구성
* **보고서 머리글/바닥글:** 보고서의 첫 페이지 상단과 마지막 페이지 하단에 한 번만 출력됩니다.
* **페이지 머리글/바닥글:** 모든 페이지의 상단과 하단에 반복 출력됩니다. (날짜, 페이지 번호 등)
* **본문:** 실제 데이터 레코드가 반복되어 출력되는 영역입니다.
* **그룹 머리글/바닥글:** 특정 필드를 기준으로 그룹화했을 때 해당 그룹의 제목이나 합계를 표시합니다. (실기 시험의 핵심)
