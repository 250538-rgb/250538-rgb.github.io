## 📍 1. 운영체제(OS) 개념

운영체제(Operating System)는 컴퓨터의 하드웨어 자원(CPU, 메모리, 디스크 등)을 효율적으로 관리하고, 사용자가 컴퓨터를 편리하고 효과적으로 사용할 수 있도록 환경을 제공하는 **가장 핵심적인 시스템 소프트웨어**입니다. (예: Windows, Linux, macOS)

### 1.1) 운영체제의 주요 기능
단순한 연결을 넘어, 컴퓨터 내부의 보이지 않는 자원들을 지휘하는 역할을 합니다.
* **프로세스 관리:** CPU의 시간을 어떤 프로그램에 먼저 할당할지 스케줄링합니다.
* **기억장치 관리:** 한정된 RAM(주기억장치) 공간을 여러 프로그램이 효율적으로 나눠 쓸 수 있도록 관리합니다.
* **입출력(I/O) 장치 관리:** 키보드, 마우스, 프린터 등 주변 장치와 프로그램 간의 데이터 입출력을 통제합니다.
* **파일 관리:** 디스크 내의 파일 생성, 삭제, 권한 부여 및 디렉토리 구조를 관리합니다.

### 1.2) 운영체제의 4대 주요 목적 
운영체제의 성능을 평가하는 기준이자, 운영체제가 궁극적으로 달성하고자 하는 목표입니다. 반드시 암기해야 합니다.

* **처리능력 (Throughput) ⬆️ 향상:** 일정 시간 동안 시스템이 처리하는 일의 양.
* **신뢰도 (Reliability) ⬆️ 향상:** 시스템이 주어진 문제를 얼마나 정확하게 해결하는가를 의미.
* **사용 가능도 (Availability) ⬆️ 향상:** 시스템을 즉시 사용할 수 있는 정도. 고장이 나더라도 빠르게 복구되어 사용할 수 있어야 함.
* **응답시간 (Turn-around Time) ⬇️ 최소화:** 시스템에 작업을 의뢰한 시간부터 처리가 완료될 때까지 걸린 시간.

> **✍️ 핵심 체크:** "운영체제의 목적은 응답시간의 최대화이다." (X) -> 응답시간은 최소화되어야 합니다.

## 📍 2. 운영체제 환경 설정

개발이나 서버 운영을 위해서는 설치된 운영체제를 용도에 맞게 설정해야 합니다.

### 환경 변수 (Environment Variable)
* 운영체제가 돌아가는 데 필요한 동적인 값들의 모임입니다.
* **`PATH` 변수:** 실행 파일이 있는 디렉토리의 경로를 지정합니다. 이 경로를 설정해두면 어느 위치에서나 해당 프로그램을 절대경로 없이 명령어만으로 실행할 수 있습니다. (예: Java, Python 등을 설치한 후 터미널에서 바로 실행하기 위해 설정)

### 사용자 및 권한 제어 환경
* 다중 사용자 환경(Linux 등)에서는 각 사용자별로 독립된 작업 공간(Home Directory)과 권한을 설정해야 시스템 보안이 유지되며, 권한 없는 사용자의 시스템 접근을 차단할 수 있습니다.

## 📍 3. 운영체제 종류 및 특성

### 1) Windows 운영체제
Microsoft에서 개발한 개인용 컴퓨터의 표준 운영체제입니다.
* **GUI (Graphic User Interface):** 키보드로 명령어를 치지 않고 마우스로 아이콘을 클릭하여 직관적으로 작업.
* **선점형 멀티태스킹 (Preemptive Multi-tasking):** OS가 CPU의 통제권을 갖고 있어, 특정 프로그램이 다운되더라도 강제로 종료시키고 다른 작업을 정상적으로 수행할 수 있음.
* **PnP (Plug and Play):** 하드웨어를 꽂기만 하면 OS가 자동으로 장치를 인식하고 드라이버를 설치해 주는 자동 감지 기능.
* **OLE (Object Linking and Embedding):** 다른 프로그램에서 작성된 개체(그림, 표 등)를 현재 작성 중인 문서에 연결하거나 삽입하는 기능.

### 2) UNIX / Linux 운영체제
서버용으로 주로 사용되며, C언어로 작성되어 이식성(다른 하드웨어로의 포팅)이 뛰어난 운영체제입니다.
* **CLI (Command Line Interface):** 터미널 창에서 텍스트 명령어를 직접 입력하여 시스템을 제어.
* **다중 사용자 (Multi-User) / 다중 작업 (Multi-Tasking):** 여러 사람이 네트워크를 통해 동시에 접속하여 여러 프로그램을 돌릴 수 있음.
* **트리(Tree) 구조의 파일 시스템:** 최상위 루트 디렉토리(`/`)를 기준으로 나뭇가지처럼 하위 디렉토리가 뻗어 나가는 계층적 파일 구조.

## 📍 4. 운영체제 명령어

서버와 PC를 직접 제어하기 위한 핵심 명령어입니다. 시험에서는 각 운영체제의 명령어 역할과 차이점을 묻는 문제가 자주 출제됩니다.

### 🪟 윈도우(Windows) 명령 프롬프트(cmd) 핵심 명령어 마스터

| 분류                 | 명령어           | 설명                                               | 사용 예시                     |
| :------------------- | :--------------- | :------------------------------------------------- | :---------------------------- |
| **파일 및 디렉토리** | `dir`            | 현재 위치의 파일 및 서브 디렉토리 목록 출력        | `dir /w` (넓게 보기)          |
|                      | `cd` (`chdir`)   | 디렉토리(폴더) 이동                                | `cd C:\Windows`               |
|                      | `md` (`mkdir`)   | 새로운 디렉토리 생성                               | `md study_folder`             |
|                      | `rd` (`rmdir`)   | 디렉토리 삭제 (비어있어야 함)                      | `rd study_folder`             |
|                      | `copy` / `xcopy` | 파일 복사 / 폴더와 하위 구조까지 모두 복사         | `xcopy C:\A D:\B /s`          |
|                      | `del` (`erase`)  | 파일 삭제                                          | `del temp.txt`                |
|                      | `ren` (`rename`) | 파일 이름 변경                                     | `ren old.txt new.txt`         |
|                      | `type`           | 텍스트 파일의 내용을 화면에 출력                   | `type readme.txt`             |
|                      | `attrib`         | 파일의 속성(읽기 전용, 숨김 등) 변경 및 확인       | `attrib +h secret.txt` (숨김) |
| **디스크 및 시스템** | `cls`            | 명령 프롬프트 화면 지우기 (Clear Screen)           | `cls`                         |
|                      | `chkdsk`         | 디스크의 논리적/물리적 오류 검사 및 복구           | `chkdsk C: /f`                |
|                      | `format`         | 디스크(파티션)를 초기화하고 파일 시스템 생성       | `format D: /fs:ntfs`          |
|                      | `tasklist`       | 현재 실행 중인 프로세스 목록 표시                  | `tasklist`                    |
|                      | `taskkill`       | 실행 중인 프로세스를 강제 종료                     | `taskkill /pid 1234 /f`       |
| **네트워크**         | `ipconfig`       | 내 컴퓨터의 IP 주소 및 네트워크 설정 확인          | `ipconfig /all`               |
|                      | `ping`           | 특정 목적지(IP/도메인)와의 네트워크 통신 상태 점검 | `ping 8.8.8.8`                |
|                      | `tracert`        | 목적지까지 패킷이 거쳐가는 라우터 경로 추적        | `tracert google.com`          |
|                      | `netstat`        | 현재 컴퓨터의 네트워크 연결 상태 및 포트 확인      | `netstat -an`                 |

### 🐧 리눅스(Linux) / 유닉스(UNIX) 핵심 명령어 마스터

| 분류                 | 명령어                | 설명                                                      | 사용 예시                         |
| :------------------- | :-------------------- | :-------------------------------------------------------- | :-------------------------------- |
| **파일 및 디렉토리** | `ls`                  | 파일/디렉토리 목록 출력                                   | `ls -al` (숨김 포함 상세출력)     |
|                      | `cd` / `pwd`          | 디렉토리 이동 / 현재 작업 중인 절대 경로 출력             | `pwd`                             |
|                      | `mkdir` / `rmdir`     | 디렉토리 생성 / 삭제                                      | `mkdir test`                      |
|                      | `cat`                 | 파일의 내용을 화면에 출력                                 | `cat readme.txt`                  |
|                      | `cp` / `mv`           | 파일 복사 / 이동 및 이름 변경                             | `cp file1 file2`                  |
|                      | `rm`                  | 파일 또는 디렉토리 삭제                                   | `rm -rf /` (위험!)                |
|                      | `tar`                 | 여러 파일을 하나의 파일로 묶거나 압축 해제                | `tar -cvf arc.tar folder/`        |
|                      | `grep`                | 파일 내에서 특정 문자열이나 패턴을 검색 (매우 중요)       | `grep "error" log.txt`            |
|                      | `find`                | 다양한 조건(이름, 크기, 권한 등)으로 파일 검색            | `find / -name "test.txt"`         |
| **권한 및 소유자**   | `chmod`               | 파일의 권한(읽기/쓰기/실행) 변경 및 특수 권한 부여        | `chmod 1777 /share` (스티키 비트) |
|                      | `chown`               | 파일의 소유자(Owner) 또는 소유 그룹 변경                  | `chown user1:group1 file.txt`     |
| **사용자/그룹 관리** | `useradd` / `adduser` | 새로운 사용자 계정 생성 (홈 디렉토리 생성 포함)           | `useradd -m user2`                |
|                      | `passwd`              | 사용자 계정의 비밀번호 설정 및 변경                       | `passwd user1`                    |
|                      | `usermod`             | 기존 사용자의 속성(그룹 등) 변경                          | `usermod -aG sudo user1`          |
|                      | `userdel`             | 사용자 계정 삭제                                          | `userdel -r user1`                |
| **프로세스/시스템**  | `ps`                  | 현재 실행 중인 프로세스의 상태 출력                       | `ps -ef`                          |
|                      | `kill`                | 프로세스 ID(PID)를 지정하여 프로세스 강제 종료            | `kill -9 1234`                    |
|                      | `top`                 | 시스템의 CPU, 메모리 사용량 및 프로세스를 실시간 모니터링 | `top`                             |
| **네트워크 관리**    | `ifconfig` / `ip`     | 네트워크 인터페이스 설정 및 IP 주소 확인                  | `ifconfig` 또는 `ip addr`         |
|                      | `ping`                | 네트워크 연결 상태 확인                                   | `ping 168.126.63.1`               |
|                      | `netstat`             | 활성화된 네트워크 연결 및 리스닝 중인 포트 확인           | `netstat -tuln`                   |
