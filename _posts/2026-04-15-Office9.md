# 📝 데이터 조회

>**과목:** 1과목. 사무자동화시스템  
**단원:** 4. SQL 작성 > 1. 데이터 조회  
**학습 목표:** 데이터베이스에서 원하는 데이터를 추출하는 SELECT 문의 기본 구조를 익히고, 조인, 집합 연산자, 내장 함수 및 데이터 사전의 개념을 이해한다.

---

## 📍 1. Select 명령어

SQL에서 데이터를 검색할 때 사용하는 가장 핵심적인 명령어입니다. 

### 1.1) SELECT 구문 구조
데이터를 조회할 때는 아래의 순서대로 명령어를 작성하며, 이 순서는 반드시 지켜야 합니다.

* **SELECT:** 조회하고자 하는 필드(속성)의 이름을 지정합니다. (모든 필드를 조회할 때는 `*` 사용)
* **FROM:** 데이터를 가져올 테이블의 이름을 지정합니다.
* **WHERE:** 조회할 데이터의 조건을 지정합니다. (예: `가격 >= 20000`)
* **ORDER BY:** 조회된 결과를 특정 필드 기준으로 정렬합니다. (`ASC` 오름차순, `DESC` 내림차순)

### 1.2) 데이터 그룹화
* **GROUP BY:** 특정 필드를 기준으로 데이터를 그룹으로 묶습니다.
* **HAVING:** `GROUP BY`로 묶인 그룹에 조건을 부여할 때 사용합니다. (`WHERE`는 개별 데이터에 조건을 걸고, `HAVING`은 그룹화된 결과에 조건을 겁니다.)

---

## 📍 2. 테이블 조인 (Join)

서로 다른 두 개 이상의 테이블을 공통된 필드를 기준으로 연결하여 하나의 결과 테이블로 만드는 기능입니다.

### 2.1) 내부 조인 (INNER JOIN)
* 두 테이블 모두에 공통으로 존재하는(교집합) 데이터만 연결하여 조회합니다.
* 가장 일반적으로 사용되는 조인 방식입니다.

### 2.2) 외부 조인 (OUTER JOIN)
* 조인 조건에 맞지 않는 데이터도 한쪽 테이블을 기준으로 모두 조회 결과에 포함시킵니다.
* **LEFT / RIGHT / FULL OUTER JOIN:** 기준이 되는 방향(왼쪽, 오른쪽, 또는 양쪽 전체)의 모든 데이터를 살리고, 매칭되지 않는 반대쪽 데이터 빈칸은 NULL로 표시합니다.

---

## 📍 3. 집합연산자

두 개 이상의 SELECT 쿼리 결과를 하나로 합치거나 비교할 때 사용하는 연산자입니다.

### 3.1) 합집합 연산
* **UNION:** 두 쿼리의 결과를 하나로 합칩니다. (중복된 레코드는 하나만 표시)
* **UNION ALL:** 두 쿼리의 결과를 합치되, 중복된 데이터도 제거하지 않고 모두 그대로 표시합니다.

### 3.2) 교집합 및 차집합 연산
* **INTERSECT (교집합):** 두 쿼리의 결과 중 공통으로 존재하는 데이터만 조회합니다.
* **EXCEPT / MINUS (차집합):** 첫 번째 쿼리의 결과에서 두 번째 쿼리의 결과와 겹치는 데이터를 제외하고 조회합니다.

## 📍 4. DBMS 내장 함수

데이터베이스 시스템(DBMS) 내부에 미리 정의되어 있어, 데이터를 가공하거나 계산할 때 편리하게 사용할 수 있는 함수들입니다.

### 4.1) 단일행 함수
각각의 데이터 행(Record)마다 적용되어 결과를 반환합니다.
* **문자열 함수:** `SUBSTR`(문자열 일부 추출), `LENGTH`(길이 반환), `UPPER/LOWER`(대소문자 변환) 등
* **숫자 함수:** `ROUND`(반올림), `TRUNC`(버림), `ABS`(절댓값) 등
* **날짜 함수:** `SYSDATE` 또는 `NOW()` (현재 시스템 날짜와 시간 반환)

### 4.2) 복수행(집계) 함수
여러 행의 데이터를 그룹으로 묶어서 하나의 연산 결과를 반환합니다. (주로 `GROUP BY`와 함께 사용)
* `SUM`(합계), `AVG`(평균), `MAX`(최댓값), `MIN`(최솟값), `COUNT`(행의 개수)

---

## 📍 5. 데이터 사전 (Data Dictionary)

데이터베이스 관리 시스템(DBMS)이 자신을 유지하고 관리하기 위해 스스로 만들어낸 아주 특별한 테이블들의 모음입니다.

### 5.1) 데이터 사전의 특징
* **메타데이터(Metadata) 저장:** 사용자 정보, 권한 내역, 생성된 테이블과 뷰의 구조, 인덱스 정보 등 '데이터를 설명하는 데이터'가 저장되어 있습니다.
* **시스템 카탈로그 (System Catalog):** 데이터 사전과 거의 같은 의미로 혼용되어 사용됩니다.
* **읽기 전용 (Read-Only):** 데이터 사전의 내용은 DBMS 시스템이 자동으로 업데이트하며, 일반 사용자는 오직 `SELECT` 문을 통해 내용을 '조회(읽기)'만 할 수 있고 임의로 수정(`UPDATE`)하거나 삭제(`DELETE`)할 수 없습니다.
