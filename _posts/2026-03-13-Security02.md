---

> **리눅스 커널과 소통하는 인터페이스인 Shell의 역할을 이해하고, 설정 파일과 해킹 로그가 어디에 쌓이는지 리눅스만의 계층적 파일 시스템(FHS) 구조를 탐색했다.**

## 💡 Why & How | 핵심 개념
* **통역사 쉘(Shell):** 내가 터미널에 명령어를 치면 쉘이 그걸 리눅스 핵심(커널)에 전달해준다는 걸 알았다. 쉘은 나랑 리눅스 사이의 필수적인 소통 창구다.
* **중요한 폴더 위치 외우기:** 리눅스는 폴더마다 역할이 딱 정해져 있다. 시스템의 두뇌인 설정 파일은 `/etc`에 있고, 내가 나중에 분석해야 할 해킹 기록(로그)은 `/var/log`에 쌓인다는 걸 확인했다.
* **길 찾기 연습:** 루트(`/`)부터 시작하는 전체 주소(절대경로)와 지금 내 위치 기준 주소(상대경로)의 차이를 배웠다. 서버를 관리할 때는 헷갈리지 않게 무조건 절대경로를 쓰는 게 안전하다는 것도 깨달았다.

## 🛠️ 오늘 사용한 핵심 명령어

```bash
# 1. 위치 파악 및 탐색 (Navigation)
pwd              # 현재 내 위치(절대경로) 확인
cd /var/log      # 특정 디렉토리로 이동
ls -al           # 숨김 파일 포함 상세 정보 확인

# 2. 파일 및 폴더 관리 (Management)
mkdir study_dir  # 새로운 폴더 생성
touch memo.txt   # 빈 파일 생성
cp memo.txt a.txt # 파일 복사
rm a.txt         # 파일 삭제

# 3. 파일 내용 열람 (Read)
cat /etc/passwd  # 파일 전체 내용 한 번에 보기
tail -f /var/log/syslog # 로그 파일 실시간 모니터링
```
