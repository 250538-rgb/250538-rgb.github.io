# 📝 데이터 수정

>**과목:** 1과목. 사무자동화시스템  
**단원:** 4. SQL 작성 > 2. 데이터 수정  
**학습 목표:** 데이터베이스의 구조를 정의하는 DDL, 데이터를 조작하는 DML, 권한을 제어하는 DCL 명령어의 종류를 구분하고 데이터 무결성 제약조건을 파악한다.

---

## 📍 1. DDL 명령어 (데이터 정의어)

데이터베이스, 테이블, 인덱스 등 데이터베이스의 전체적인 뼈대(구조)를 만들고 변경하며 삭제하는 명령어입니다.

### 1.1) CREATE (생성)
* 새로운 테이블이나 데이터베이스 구조를 생성합니다.
* `CREATE TABLE 테이블명 (필드명 데이터타입, ...);`

### 1.2) ALTER (변경)
* 기존에 만들어진 테이블의 구조(새로운 필드 추가, 기존 필드 삭제 등)를 변경합니다.
* `ALTER TABLE 테이블명 ADD 필드명 데이터타입;`

### 1.3) DROP (삭제)
* 테이블 구조 자체를 완전히 삭제합니다. (내부에 있던 데이터도 모두 영구적으로 소멸됩니다.)
* `DROP TABLE 테이블명;`

---

## 📍 2. Insert, Delete, Update 명령어 (데이터 조작어, DML)

테이블 안에 들어있는 실제 데이터(레코드)를 삽입, 수정, 삭제하는 명령어입니다.

### 2.1) INSERT (데이터 삽입)
* 테이블에 새로운 데이터 행을 추가합니다.
* `INSERT INTO 테이블명 (필드1, 필드2) VALUES (값1, 값2);`

### 2.2) UPDATE (데이터 수정)
* 테이블에 이미 저장된 데이터의 값을 변경합니다.
* `UPDATE 테이블명 SET 필드명 = 변경할값 WHERE 조건;`
* *(주의: `WHERE` 조건을 지정하지 않으면 테이블의 모든 데이터가 한꺼번에 변경됩니다.)*

### 2.3) DELETE (데이터 삭제)
* 테이블에서 특정 조건에 맞는 데이터 행만 삭제합니다.
* `DELETE FROM 테이블명 WHERE 조건;`

## 📍 3. DCL 명령어 (데이터 제어어)

데이터베이스 사용자에게 권한을 주거나 뺏고, 데이터의 변경 사항을 시스템에 최종 반영하거나 취소하는 명령어입니다.

### 3.1) 권한 관리 (GRANT / REVOKE)
* **GRANT:** 사용자에게 특정 작업(조회, 수정 등)에 대한 권한을 부여합니다.
* **REVOKE:** 사용자에게 부여했던 권한을 다시 회수(취소)합니다.

### 3.2) 트랜잭션 관리 (COMMIT / ROLLBACK)
* **COMMIT:** 데이터의 변경(INSERT, UPDATE, DELETE) 작업이 성공적으로 끝났음을 데이터베이스에 영구적으로 반영합니다.
* **ROLLBACK:** 데이터 변경 작업 중 오류가 발생했거나 취소하고 싶을 때, 변경 작업 이전의 원래 상태로 되돌립니다.

---

## 📍 4. 데이터 무결성 (Data Integrity)

데이터베이스에 저장된 데이터가 항상 정확하고 일관된 상태를 유지하도록 보장하는 핵심 규칙입니다.

### 4.1) 개체 무결성 (Entity Integrity)
* 모든 테이블은 반드시 **기본키(Primary Key)**를 가져야 합니다.
* 기본키는 중복된 값을 가질 수 없고, 비어있어서도 안 된다(NULL 불가)는 규칙입니다.

### 4.2) 참조 무결성 (Referential Integrity)
* 두 테이블이 서로 관계가 맺어져 있을 때, 참조하는 **외래키(Foreign Key)**의 값은 반드시 참조 당하는 기본 테이블에 실제로 존재하는 값이어야 한다는 규칙입니다.
* 부모 테이블에 없는 데이터를 자식 테이블에 마음대로 입력할 수 없도록 막아줍니다.
