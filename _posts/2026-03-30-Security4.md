# 📖 오늘 배운 내용

---

> **수만 줄의 서버 접속 기록에서 `grep`과 정규표현식으로 단서를 찾고, `sed`, `awk`, `sort` 등 다양한 텍스트 처리 도구를 파이프(`|`)로 연결해 나만의 로그 분석 파이프라인을 구축했다.**

## 💡 Why & How | 핵심 개념
* **터미널의 Ctrl+F, `grep`:** 윈도우의 찾기가 단순 단어 매칭이라면, `grep`은 정규표현식을 이용해 원하는 정보만 족집게처럼 뽑아내는 강력한 검색 엔진이다.
* **정규표현식(Regex) 4대장:** '특정 패턴'을 찾는 마법의 기호들이다.
    * `^` (시작): `^root` (root로 시작하는 줄)
    * `$` (끝): `bash$` (bash로 끝나는 줄)
    * `.` (임의의 한 글자): `a.c` (a와 c 사이 아무 글자 하나)
    * `*` (앞 글자 반복): `.*` (모든 문자열)
* **텍스트 처리의 조립 라인 (도구 모음):** 리눅스 명령어의 진가는 파이프(`|`)로 연결할 때 나타난다.
    * `sed`, `tr`: 글자를 통째로 바꾸거나 지우는 조각칼.
    * `awk`, `cut`: 띄어쓰기나 기호(:)를 기준으로 필요한 '칸(Column)'만 쏙 뽑아내는 정밀한 집게.
    * `sort`, `uniq`, `wc`: 데이터를 정렬하고, 중복을 제거하며, 전체 개수를 세는 통계의 핵심.
* **결과 저장하기:** 터미널 출력에 그치지 않고 리다이렉션(`>>`)을 통해 알짜배기 정보를 파일로 안전하게 기록했다.

## 🛠️ 오늘 사용한 핵심 명령어

```bash
# 1. 정규표현식을 활용한 고급 검색 (grep)
grep "^root" /etc/passwd         # 'root'로 시작하는 줄만 검색
grep "false$" /etc/passwd        # 'false'로 끝나는 줄만 검색
grep "Fail.*root" /var/log/auth.log # 'Fail'로 시작해 'root'가 들어간 줄 검색

# 2. 텍스트 필터링 및 수정 (sed, tr)
sed -i 's/old/new/g' file.txt    # 파일 내 글자 한꺼번에 바꾸기
sed '3d' file.txt                # 3번째 줄만 삭제해서 보여주기
tr 'a-z' 'A-Z' < file.txt        # 소문자를 대문자로 변환 (치환)

# 3. 필요한 데이터만 추출 (awk, cut)
awk '{print $1, $11}' auth.log   # 띄어쓰기 기준 1번째, 11번째 칸만 추출
cut -d ':' -f 1 /etc/passwd      # 구분자(:) 기준 1번째 칸(계정명)만 자르기

# 4. 데이터 정리 및 통계 (sort, uniq, wc)
sort log.txt | uniq              # 정렬 후 중복 제거
sort log.txt | uniq -c           # 중복된 개수까지 세기 (통계 핵심)
wc -l log.txt                    # 전체 줄 수(Line count) 세기

# 5. [실전 예제] 파이프라인 완성: 가장 많이 접속 실패한 해커 IP Top 5 뽑기
cat /var/log/auth.log | grep "Failed" | awk '{print $11}' | sort | uniq -c | sort -rn | head -n 5
# (해석: 로그 열기 -> 실패 찾기 -> IP칸 추출 -> 정렬 -> 중복개수 세기 -> 큰 숫자순 정렬 -> 상위 5개)
```
